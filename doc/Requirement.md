Project: React-Based Recipe Finder Prototype (Mobile-Responsive)
Overview
I am validating a concept for an interactive recipe discovery platform. Before investing in a full
backend, I need a single web developer to build a lightweight, mobile-responsive React
prototype. The application will allow users to input ingredients they have on hand and instantly
view recipe suggestions fetched from a public API.
Scope of Work
• Frontend Framework: Develop a Single Page Application (SPA) using React (via Vite
or Create React App).
• API Integration: Connect directly to the free tier of the Spoonacular API or Edamam
API to handle all recipe logic and data fetching.
• Core Feature - Ingredient Search: Build a clean search bar where users can type
comma-separated ingredients (e.g., "chicken, rice, broccoli") to query the API.
• Results Grid: Display the API responses in a responsive, mobile-friendly grid of "Recipe
Cards" showing the recipe image, title, and total cooking time.
• Recipe View: Clicking a recipe card should link the user directly to the external source
URL provided by the API (no custom step-by-step instruction pages needed for this
phase).
• Dietary Filters: Implement 2-3 hardcoded toggle buttons (e.g., Vegan, Gluten Free,
Peanuts) that append standard parameters to the API request.
• Deployment: Deploy the live prototype to a free hosting service like Vercel, Netlify, or
GitHub Pages.
Exclusions (Do NOT build these):
• No custom backend, databases, or user authentication.
• No machine learning or "smart" recommendation engines.
• No drag-and-drop mechanics or custom recipe creation tools.
• No shopping list exports or local analytics tracking.
Deliverables
1. A link to the live, hosted React prototype.
2. A zip file or GitHub repository link containing the clean, commented React source code.
3. A brief text document explaining where and how to swap out the free API key.
Design & Resources
• APIs: Spoonacular (https://spoonacular.com/food-api) or Edamam
(https://developer.edamam.com/edamam-docs-recipe-api).
• Typography: Google Fonts (Poppins for headings, Open Sans for body).
• Color Palette: #264653, #2a9d8f, #e9c46a, #f4a261, #e76f51.
• UI Inspiration: Clean, modern cards similar to Yummly or Epicurious
